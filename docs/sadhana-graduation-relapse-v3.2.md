# Sadhana: Graduation & Relapse Protocol
## Design System Addendum v3.2

**Scope:** End-game journey, Sovereignty Audit, Post-graduation safety net  
**Prerequisite:** Phase 7 complete, Stability ≥ 90 for 30+ days  
**Core Truth:** The goal is to make the app obsolete. The OS migrates from phone to nervous system.

---

# PART I: THE GRADUATION JOURNEY

## 1.1 Philosophy

Sadhana succeeds when the user leaves.

Most apps optimize for retention. Sadhana optimizes for **internalization** — the point where the user can:
- Generate stability without prompts
- Navigate quadrants without logging
- Access Yellow (and beyond) without guardrails
- Return to Green from any state using internal resources

**The Paradox:** The better we build, the faster they leave.

---

## 1.2 Graduation Eligibility

### Hard Requirements

| Requirement | Threshold | Verification |
|-------------|-----------|--------------|
| Stability Score | ≥ 90 | System-calculated |
| Duration at HV-Eligible | 30 consecutive days | No breaks |
| Manual grounding prompts | Zero in final 30 days | User maintained without app intervention |
| Phase 7 sessions | 10+ clean Serpent sessions | No kill-switch triggers |
| Dissociation markers | Zero flags in final 60 days | All Neti re-entries clean |
| Mission completion | Weaver complete (all pillars) | Integrity Map balanced |

### Soft Indicators (Tracked, Not Gated)

| Indicator | Signal | What It Means |
|-----------|--------|---------------|
| App opens decreasing | Week-over-week decline | Less dependency |
| Session duration shortening | Practices complete faster | Internalized protocols |
| Self-corrections increasing | User chooses ground before prompted | Anticipatory awareness |
| Mood volatility decreasing | Smaller swings, faster recovery | Regulated system |

# PART III: RELAPSE PROTOCOL

## 3.1 Philosophy

Graduation doesn't mean permanent immunity.

Life delivers shocks: loss, trauma, illness, crisis. A graduated user may need to re-enter the system. This is not failure — it's appropriate use of a tool.

**The principle:** Fast re-entry, graduated protection, accelerated return to sovereignty.

---

## 3.2 Relapse Detection

### Trigger Conditions

| Trigger | Source | Response |
|---------|--------|----------|
| Red/Blue quadrant logged | User check-in | Prompt Rapid Re-Calibration |
| HRV < 50% of graduation baseline | Sensor (if available) | Prompt Rapid Re-Calibration |
| Sleep crash pattern (2 nights) | Sensor or user report | Prompt Rapid Re-Calibration |
| User requests re-entry | Explicit action | Begin Rapid Re-Calibration |

Graduated users who successfully re-calibrate have a **shortened** path back to sovereignty:

**Rationale:** The internal pathways are already established. Re-calibration is clearing a temporary block, not building new capacity.
